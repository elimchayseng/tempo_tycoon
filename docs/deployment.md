# Deployment

## Railway

The project is deployed on Railway with the following setup:

### Configuration

1. Create a new Railway project and link the GitHub repository
2. Set build command: `npm run build`
3. Set start command: `npm run start`
4. Add all required environment variables (see [Configuration](./configuration.md))

### Required Environment Variables

```
PORT=4000
NODE_ENV=production
ADMIN_TOKEN=<strong-random-token>
RPC_URL=https://rpc.moderato.tempo.xyz
ZOO_SIMULATION_ENABLED=true
LOG_LEVEL=info
```

> **Note:** `ADMIN_TOKEN` is required in production — the server will refuse to start without it. No private keys are needed — wallets are generated automatically each simulation run. See [Security](./security.md) for the full security posture.

### Health Checks

Railway can use the health endpoint for monitoring:

- **Health check path:** `/api/health`
- **Blockchain check:** `/api/health/blockchain`
- **Zoo-specific:** `/api/zoo/health`

### Verification

After deployment:

```bash
# Basic health
curl https://your-app.railway.app/api/health

# Blockchain connectivity
curl https://your-app.railway.app/api/health/blockchain

# Zoo status
curl https://your-app.railway.app/api/zoo/status

# Pre-flight checks (requires auth in production)
curl -X POST https://your-app.railway.app/api/zoo/preflight \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

### Notes

- Railway handles host binding automatically — the server binds to `0.0.0.0` on the configured `PORT`
- The `dist/` directory is generated by `npm run build` (Vite) and served as static files
- WebSocket connections are supported on Railway's default proxy
- No external databases (PostgreSQL, Redis) are required — all state is in-memory
